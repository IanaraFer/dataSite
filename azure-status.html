☁️ Azure Deployment Status

Current Status

Azure deployment is configured and ready. You just need to:

  1. Create Azure Static Web App
  2. Get deployment token
  3. Add token to GitHub Secrets
  4. Push to main branch (automatic deployment)
1

Create Azure Static Web App

Go to Azure Portal and create a new Static Web App

Open Azure Portal
2

Configure Static Web App

Use these settings when creating your Static Web App:

  • Resource Group: rg-analyticacore-prod
  • Name: analyticacore-ai (or any unique name)
  • Plan: Free
  • Region: East US 2
  • Source: GitHub
  • Repository: dataSite
  • Branch: main
  • App location: /
  • Output location: /website
3

Get Deployment Token

After creating the Static Web App, get the deployment token:

  1. Go to your Static Web App in Azure Portal
  2. Click "Manage deployment token" in the left menu
  3. Copy the deployment token
4

Add Token to GitHub Secrets

Add the deployment token to your GitHub repository secrets:

Open GitHub Secrets

Secret Name: AZURE_STATIC_WEB_APPS_API_TOKEN

Secret Value: [Your deployment token from step 3]

5

Check Deployment Status

Monitor your deployment progress:

View GitHub Actions

Look for "Azure Deployment" workflow - it should run automatically after you add the token.

Troubleshooting

  • Token not working? Make sure the secret name is exactly: AZURE_STATIC_WEB_APPS_API_TOKEN
  • Workflow not running? Push a small change to trigger it
  • Site not loading? Wait 2-3 minutes after deployment completes
  • Build errors? Check the GitHub Actions logs for specific error messages
